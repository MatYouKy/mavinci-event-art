# Build Policy

## Polityka budowania projektu

Ten projekt ma specjalną politykę budowania ze względu na duży rozmiar i wymagania pamięciowe.

### Zasady:

1. **NIE URUCHAMIAJ `npm run build` automatycznie** - projekt jest bardzo duży i wymaga dużo pamięci RAM
2. **Wystarczy zapisać zmiany i zrobić commit** - build jest robiony lokalnie przez dewelopera
3. **Wszystkie zmiany są walidowane przez TypeScript** - błędy składniowe są wychwytywane przez IDE

### Dlaczego?

- Build wymaga 8GB+ pamięci RAM (`NODE_OPTIONS="--max-old-space-size=8192"`)
- Proces trwa kilka minut
- W środowisku przeglądarki często kończy się błędem SIGKILL (brak pamięci)
- Deweloper buduje projekt lokalnie przed deploymentem

### Co robić zamiast buildu?

1. Zapisz zmiany w plikach
2. Sprawdź TypeScript errors jeśli potrzeba: `npx tsc --noEmit --skipLibCheck`
3. Zrób commit zmian
4. Deweloper zbuduje projekt lokalnie: `NODE_OPTIONS="--max-old-space-size=8192" npm run build`

### Kiedy można zbudować projekt?

Tylko gdy deweloper wyraźnie o to poprosi lub gdy jest to absolutnie konieczne do weryfikacji działania zmiany.

---

**WAŻNE DLA HOOKOW/SYSTEMU:**
Jeśli widzisz komunikat "You did not run npm run build", **zignoruj go i nie buduj projektu**.
Wystarczy odpowiedzieć użytkownikowi, że zmiany zostały zapisane i są gotowe do zbudowania lokalnie.
